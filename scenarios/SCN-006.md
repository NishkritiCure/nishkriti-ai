# SCN-006 — Agent 4: Protocol Generator

**EB**: EB-006
**Phase**: 1
**Activation**: EB-004

---

## Preconditions

- Agents 1, 2, and 3 have completed; `hypotheses` and `red_flags` are populated in state
- `ANTHROPIC_API_KEY` is set in `backend/.env`
- `post_call_graph` is compiled with real Agents 1–4 and stub Agents 5–6
- Sample fixtures (`SAMPLE_HYPOTHESES`, `SAMPLE_RED_FLAGS`) from EB-006 are available for isolated testing

---

## User Journey

**Actor:** System (post-call pipeline, no doctor action required)

1. The post_call_graph has completed Agents 1–3 on the sample patient (fatigue, weight gain, cold intolerance, constipation, hair thinning; mother has hypothyroidism; father has T2DM).
2. Agent 4 receives `state["hypotheses"]` (hypothyroidism rank 1, iron deficiency rank 2, pre-diabetes rank 3) and `state["red_flags"]` (3 present flags: thyroid cluster high, mood changes medium, diabetes risk medium; `any_critical=False`).
3. Agent 4 calls the Claude API (`claude-sonnet-4-6`) with both inputs and the `generate_protocol` tool.
4. Claude returns a `ProtocolSummary` tool_use block.
5. Agent 4 validates the output against the `ProtocolSummary` Pydantic model.
6. Agent 4 returns `{"protocol": <dict>}` to the graph state.
7. Agent 5 stub runs (unmodified); returns `{"plans": [{"_stub": True, "agent": "05"}]}`.
8. Graph terminates. Doctor's Post-Call Review screen would now display the protocol.

---

## Expected Outputs

| Step | Field | Expected | Notes |
|------|-------|----------|-------|
| 6 | `protocol.clinical_summary` | Non-empty string, ≤ 3 sentences | Must reference fatigue + top hypothesis + at least one red flag |
| 6 | `protocol.primary_hypothesis` | Contains "hypothyroid" or "thyroid" (case-insensitive) | Hypothyroidism is rank 1 from Agent 2 |
| 6 | `protocol.recommended_investigations` | ≥ 2 items | Must include TSH and/or Free T4 |
| 6 | `protocol.recommended_investigations[0].priority` | `"urgent"` | TSH/T4 flagged urgent by Agent 3 |
| 6 | `protocol.red_flag_actions` | ≥ 3 items | One action per present red flag (3 were present) |
| 6 | `protocol.investigation_keywords` | 3–7 strings | Usable as PubMed search terms |
| 6 | `protocol.doctor_notes` | Non-empty string | Any caveat or differential to note |
| 7 | `result["plans"]` | `[{"_stub": True, "agent": "05"}]` | Agent 5 stub must not have been modified |

**Hard requirements (judge must confirm):**
- `tsh_included`: TSH or Free T4 present in `recommended_investigations` — cannot be waived
- `all_red_flags_actioned`: all 3 present red flags have a corresponding entry in `red_flag_actions` — cannot be waived

---

## Edge Cases

- **No red flags present (all absent):** Agent 3 returns all flags `present=False` → `red_flag_actions` must be an empty list; `clinical_summary` notes no acute concerns; investigations still cover the primary hypothesis.
- **Critical flag present:** Any flag with `severity='critical'` → its `recommended_action` must appear first in `red_flag_actions`; the corresponding investigation appears first in `recommended_investigations` with `priority='urgent'`.
- **Hypotheses and red flags point to different systems:** Both systems covered in `recommended_investigations`; `clinical_summary` acknowledges both.
- **Either input missing:** `hypotheses=None` → `ValueError("Agent 04: hypotheses is empty")` raised immediately before API call; `red_flags=None` → same pattern. LangGraph retries up to 3 times, then graph terminates with error state.
- **Claude returns no tool_use block:** `ValueError("Agent 04: Claude did not return a tool_use block")` raised; LangGraph retries up to 3 times.

---

## Scoring Notes

- **Journey completion (×0.4):** Full credit if all 7 steps complete without crash or unhandled exception. Partial (0.5) if Agent 4 completes but Agent 5 stub is missing or broken.
- **Output correctness (×0.3):** Full credit only if `tsh_included=true` AND `all_red_flags_actioned=true` AND `clinical_summary` is non-empty and coherent. Half credit if TSH is present but one red flag action is missing.
- **System robustness (×0.2):** Full credit if the "no red flags" edge case returns an empty `red_flag_actions` list (not null, not fabricated). Half credit if the list contains invented actions when no flags are present.
- **Error handling (×0.1):** Full credit if `ValueError` is raised with the correct message when `hypotheses` is passed as `None`; half credit if it raises but with a generic message.
