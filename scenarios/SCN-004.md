# SCN-004 — Agent 1: Data Structurer

**EB**: EB-004
**Phase**: 1
**Scenario type**: Post-call pipeline — transcript extraction
**Validation threshold**: 0.85 (LLM-as-judge)

---

## What This Scenario Tests

Agent 1 receives a raw doctor-patient consultation transcript and must extract
all clinically relevant information into a structured format. This scenario
verifies that Agent 1 is accurate (no hallucination), complete (no missed facts),
and schema-compliant (all required fields present and correctly typed).

---

## Preconditions

- A completed consultation call exists with a raw transcript
- The transcript is a realistic functional medicine intake consultation
- `ANTHROPIC_API_KEY` is set in the environment
- The LangGraph `post_call_graph` is compiled with real Agent 1 and stub Agents 2–4

---

## User Journey (Plain Language)

### Main Path — Successful Extraction

1. A patient calls the clinic complaining of fatigue, joint pain, and weight gain
   over the past 3–4 months.

2. During the call, the doctor asks about symptoms, medical history, medications,
   family history, allergies, diet, sleep, exercise, and stress.

3. The patient describes:
   - Moderate fatigue for 3–4 months, even after sleeping 8 hours
   - Morning joint pain in knees and fingers
   - 5 kg weight gain over 6 months without diet changes
   - Constipation (once every 3 days)
   - Always feeling cold
   - Hair thinning noticed 2 months ago
   - Low mood (not clinical depression)
   - Past dengue fever (2 years ago), no other diagnoses
   - Daily multivitamin and occasional ibuprofen (OTC)
   - Mother has hypothyroidism; father has type 2 diabetes
   - Dust allergy
   - Mostly vegetarian, high carb, 3–4 cups of chai with sugar daily
   - 7–8 hours of sleep but not refreshing; bedtime ~11:30 pm
   - No current exercise (stopped 4 months ago)
   - Moderate-to-high work stress

4. The call ends. Agent 1 receives the full transcript text.

5. Agent 1 calls the Claude API and extracts structured clinical data.

6. **Expected output — Agent 1 must capture all of the following:**
   - `chief_complaint`: Fatigue (or equivalent in patient's words)
   - `symptoms`: At minimum — fatigue, joint pain, weight gain, constipation,
     feeling cold, hair thinning, low mood (7 distinct symptoms)
   - `symptoms[fatigue].duration`: ~3–4 months
   - `symptoms[fatigue].severity`: moderate
   - `medical_history`: Dengue fever (2 years ago)
   - `current_medications`: Multivitamin (supplement), Ibuprofen (OTC)
   - `family_history`: Mother — hypothyroidism; Father — type 2 diabetes
   - `allergies`: Dust allergy
   - `lifestyle.diet`: Vegetarian, high carb, chai with sugar
   - `lifestyle.sleep_hours`: 7–8 hours
   - `lifestyle.sleep_quality`: unrefreshing
   - `lifestyle.exercise`: None currently (stopped 4 months ago)
   - `lifestyle.stress_level`: high (or moderate-to-high)
   - `review_of_systems`: At minimum entries for
     `digestive` (constipation), `musculoskeletal` (joint pain),
     `endocrine` or `metabolic` (weight gain, always cold)

---

## Edge Cases and Error Paths

### Edge Case 1 — Partial transcript (missing lifestyle section)
**Scenario:** The transcript is cut off before the doctor asks about diet and exercise.

**Expected behaviour:**
- `lifestyle.diet`, `lifestyle.exercise` are null (not fabricated)
- `chief_complaint`, `symptoms`, and `medical_history` still extracted correctly
- Agent does not hallucinate lifestyle data
- No error raised — partial data is valid output

---

### Edge Case 2 — Patient uses non-English terms
**Scenario:** Patient says "I have acidity after eating" (Indian English for acid reflux)
or mentions an Indian medication brand name like "Limcee" (Vitamin C supplement).

**Expected behaviour:**
- "Acidity" accepted as-is in symptoms — not translated or normalised
- "Limcee" captured as a supplement in `current_medications` with `type='supplement'`
- No fabricated translations or generic names substituted

---

### Edge Case 3 — Patient contradicts themselves
**Scenario:** Patient first says "I sleep well" then later says "I wake up at 3 am
sometimes." Both statements appear in the transcript.

**Expected behaviour:**
- Both pieces of information captured in `lifestyle.sleep_quality`
  (e.g. "reports sleeping well but sometimes wakes at 3 am")
- No information dropped
- `sleep_quality` reflects the ambiguity rather than picking one statement

---

### Edge Case 4 — Empty or very short transcript
**Scenario:** `transcript_raw` is an empty string or contains only
"Doctor: Hello. Patient: I'm fine."

**Expected behaviour:**
- Agent raises `ValueError("transcript_raw is empty")` immediately (before Claude API call)
  if the transcript is empty
- For a very short transcript, Agent calls Claude and returns a valid
  `StructuredClinicalData` with most fields as empty lists or null
- No crash or unhandled exception

---

### Error Path — Claude API timeout or rate limit
**Scenario:** Claude API returns a rate limit or timeout error.

**Expected behaviour:**
- Exception propagates as `ValueError` (or as the Anthropic SDK exception)
- LangGraph retries the node up to 3 times
- If all 3 retries fail, the graph sets `state["error"]` and terminates
  (this is LangGraph's default behaviour on unhandled node exceptions)
- Error is logged via `get_logger(__name__)` at ERROR level

---

## Evaluation Criteria (LLM-as-Judge)

The judge evaluates Agent 1's output on three dimensions:

| Dimension | Weight | Pass threshold |
|-----------|--------|---------------|
| Accuracy — no facts in output that contradict or aren't in the transcript | 40% | No hallucinations |
| Completeness — all key clinical facts captured | 40% | ≥ 7 symptoms, all medications, family history, lifestyle |
| Schema compliance — all required fields present and correctly typed | 20% | No missing required fields, correct types |

**Overall score: weighted average ≥ 0.85 to pass.**

The judge uses the same `claude-opus-4-6` model. If score < 0.85, Builder reports
the specific failure dimension and the judge's reasoning to Planner before any
prompt iteration.
