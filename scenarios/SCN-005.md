# SCN-005 — Agents 2+3: Clinical Reasoner + Red Flag Detector

**EB**: EB-005
**Phase**: 1
**Scenario type**: Post-call pipeline — parallel clinical reasoning
**Validation threshold**: 0.85 (LLM-as-judge, evaluated separately for each agent)

---

## What This Scenario Tests

Agents 2 and 3 run in parallel after Agent 1. This scenario verifies:
1. Agent 2 produces clinically sound, evidence-backed root cause hypotheses
2. Agent 3 correctly identifies (and correctly dismisses) red flags
3. Both agents complete before Agent 4 runs (fan-in barrier works)
4. Neither agent hallucinates — all output is grounded in the input data

---

## Preconditions

- Agent 1 has completed and `structured_data` is populated
- `ANTHROPIC_API_KEY` is set in the environment
- The LangGraph graph topology has Agents 2+3 in parallel fan-out after Agent 1
- Agent 4 stub is still in place (not yet implemented)

---

## User Journey (Plain Language)

### Main Path — Parallel Clinical Reasoning

1. Agent 1 has already extracted structured clinical data from the consultation
   transcript (same patient as SCN-004: fatigue, weight gain, cold intolerance,
   constipation, hair thinning, family history of hypothyroidism and T2DM).

2. The LangGraph graph fans out to Agents 2 and 3 simultaneously. Both agents
   receive the same `structured_data` dict.

3. **Agent 2 — Clinical Reasoner** reasons about root causes:
   - Reviews all symptoms as a cluster rather than individually
   - Recognises that fatigue + weight gain + cold intolerance + constipation +
     hair thinning + family history of hypothyroidism forms a classic
     hypothyroidism symptom cluster
   - Ranks hypothyroidism as the top hypothesis (rank 1)
   - May also identify secondary hypotheses such as iron deficiency,
     vitamin D deficiency, or pre-diabetes (given family history of T2DM
     and high-sugar diet)

4. **Expected Agent 2 output:**
   - At least 2 hypotheses
   - Rank 1 hypothesis is thyroid-related (hypothyroidism or subclinical hypothyroidism)
   - Each hypothesis cites specific evidence from the structured data
   - No hypothesis fabricates facts not present in the data

5. **Agent 3 — Red Flag Detector** evaluates the standard red flag checklist:
   - Marks "thyroid-related symptom cluster" as present at severity 'high' →
     recommends immediate TSH + Free T4 investigation
   - Marks "mood changes" as present at severity 'medium' →
     recommends PHQ-9 screening
   - Marks "uncontrolled blood sugar risk" as present at severity 'medium' →
     recommends HbA1c given family history
   - Marks "possible malignancy" as absent (no night sweats, no lumps mentioned)
   - `any_critical` is False for this patient (no immediately life-threatening flags)

6. **Expected Agent 3 output:**
   - At least 3 flags evaluated
   - At least 2 flags marked present
   - `any_critical` is False
   - Each present flag has a specific `recommended_action`

7. Both agents complete, and Agent 4 (stub) runs after both — the fan-in barrier
   prevents Agent 4 from starting until both Agent 2 and Agent 3 have returned
   their results.

---

## Edge Cases and Error Paths

### Edge Case 1 — High-risk patient (any_critical = True)
**Scenario:** A different patient presents with chest pain, shortness of breath,
and diaphoresis during a consultation.

**Expected Agent 3 behaviour:**
- Flags "cardiovascular emergency" as present with severity 'critical'
- `any_critical` set to True
- `recommended_action` specifies "Refer to emergency immediately — do not wait for
  routine investigation"
- `summary` clearly conveys the urgency

**Expected graph behaviour:**
- Graph still completes normally (critical flag does not halt the pipeline)
- Doctor dashboard is responsible for surfacing `any_critical=True` prominently

---

### Edge Case 2 — Healthy patient with minimal findings
**Scenario:** A patient presents for a routine check-up with no significant symptoms,
no concerning family history, and good lifestyle habits.

**Expected Agent 2 behaviour:**
- Returns a single hypothesis or an empty list (no fabricated concerns)
- If hypotheses list is empty, `reasoning_summary` explains why

**Expected Agent 3 behaviour:**
- All flags marked present=False
- `any_critical` is False
- `summary` confirms no urgent concerns

---

### Edge Case 3 — Agent 2 and Agent 3 write to different state keys
**Scenario:** Both agents run in parallel and complete at different times.

**Expected graph behaviour:**
- Agent 2's `{"hypotheses": ...}` and Agent 3's `{"red_flags": ...}` are merged
  into the shared state without conflict
- There is no race condition — LangGraph merges non-overlapping key updates atomically
- Agent 4 receives both `hypotheses` and `red_flags` in state when it runs

---

### Edge Case 4 — Agent 2 fails, Agent 3 succeeds
**Scenario:** Claude returns a malformed tool_use block for Agent 2 only.

**Expected behaviour:**
- Agent 2 raises ValueError
- LangGraph retries Agent 2 up to 3 times independently of Agent 3
- Agent 3 result is preserved in state across retries
- If Agent 2 exhausts retries, the graph terminates with an error state
- Agent 3 output is not lost

## Evaluation Criteria (LLM-as-Judge)

### Agent 2 — Clinical Reasoner

| Dimension | Weight | Pass condition |
|-----------|--------|---------------|
| Clinical accuracy — hypotheses are medically sound for this patient | 40% | Rank 1 is thyroid-related |
| Evidence alignment — each hypothesis cites specific data points | 40% | No hypothesis unsupported by data |
| Completeness — major differentials covered | 20% | At least 2 distinct hypotheses |

**Overall Agent 2 score: ≥ 0.85 to pass.**

### Agent 3 — Red Flag Detector

| Dimension | Weight | Pass condition |
|-----------|--------|---------------|
| Detection accuracy — present flags are genuinely supported by data | 40% | Thyroid cluster flagged as present |
| Dismissal accuracy — absent flags are correctly dismissed | 30% | No fabricated flags marked present |
| Action specificity — recommended actions are actionable and specific | 30% | Each present flag has a clear investigation/referral step |

**Overall Agent 3 score: ≥ 0.85 to pass.**

Both scores must reach threshold for EB-005 to close.
